## 📖 项目简介

`titanwings/colleague-skill` 是一个微信对话机器人的**技能/插件系统**，属于「数字生命 1.0」(Digital Life) 项目的核心组件之一。

### 核心理念

项目的诞生源于一个美好的愿望：**将冰冷的离别化为温暖的 Skill**。

在日常使用微信对话机器人的过程中，用户经常会面临这样的困境：当常用的 AI 助手下线、服务商跑路或服务终止时，用户积累的对话数据、个性化配置、模型偏好等信息都会随之消失，造成不可挽回的损失。

「数字生命 1.0」的设计初衷正是要解决这一痛点：通过**插件化架构**和**本地优先策略**，确保只要用户不主动删除 Skill 的代码和配置，AI 助手将永远在线、永远属于用户。

## ✨ 技术亮点

### 1. 插件化架构设计 ⭐⭐⭐⭐⭐

项目的核心亮点在于**标准化的 Skill 基类接口设计**：

```python
class Skill:
    """技能基类 - 所有自定义技能都需要继承此类"""
    
    def __init__(self):
        self.name = "base_skill"
        self.version = "0.0.1"
    
    # 开发者只需实现以下方法即可创建自定义技能
    def on_load(self):
        """技能加载时调用"""
        pass
    
    def on_message(self, message):
        """收到消息时调用"""
        pass
    
    def on_unload(self):
        """技能卸载时调用"""
        pass
```

**价值**：开发者无需了解框架内部实现，只需继承基类并实现业务逻辑，即可实现技能的热插拔。

### 2. 本地优先策略 ⭐⭐⭐⭐

```
数据存储策略：
├── 使用 SQLite (aiosqlite) 进行本地持久化
├── 配置文件存储在 ~/.colleague_skill/
├── 不依赖远程服务端
└── 用户真正拥有数据控制权

优势：
├── 数据不随服务消失
├── 离线可访问
└── 隐私数据不经过第三方服务器
```

### 3. 异步架构设计 ⭐⭐⭐⭐

项目采用 `aiosqlite` 实现异步数据库操作：

```python
import aiosqlite

async def get_user_data(user_id: int):
    async with aiosqlite.connect('colleague.db') as db:
        cursor = await db.execute(
            "SELECT * FROM users WHERE id = ?", 
            (user_id,)
        )
        return await cursor.fetchone()
```

**价值**：异步 I/O 设计使其能够高效处理高并发场景，适合作为微信机器人的后端服务。
